Prompt caching
cheat sheet

cache_control on X = create/check one cumulative prefix ending at X.

Wire order

tools → system → messages

A change invalidates its section and every section to its right.

Modes

AutomaticTop-level marker; Anthropic targets the last eligible block and advances it as history grows.
ExplicitBlock-level markers; you choose up to four prefix endpoints.
CombinedAllowed; the automatic endpoint consumes one of four slots.

Lifecycle

miss → process prefix → write when response begins
hit → read prefix cheaply → process new suffix → refresh TTL

Default TTL is 5 minutes; 1 hour is available at a higher write price. The cache is only usable after the first response begins.

Growing chat

R1: T + S + U1 ◀ write
R2: [T + S + U1] + A1 + U2 ◀ read + delta write

An assistant answer becomes cacheable only when it returns as input in a later request.

Alfred-specific

  • Active tools are alphabetically sorted before serialization.
  • Explicit endpoints: last tool, system, last message, and sometimes the message before a trailing tool-result burst.
  • Alfred's current client-side loaded tool changes the earliest prefix and causes one cold transition.
  • Anthropic-native defer_loading is different: discovered schemas are inserted later as tool_reference blocks, preserving the early prefix.
  • The deferred-tool “catalog” has no public resource ID, TTL, or invalidation API. Resend all definitions every request and treat that request’s tools array as authoritative.
  • Keep prompt-cache lifetime separate from tool-search lifetime: prompt entries have cache TTLs; the searchable tool view is defined anew by request input.
  • Changing runtime time stays in a later suffix, preserving stable endpoints.
  • Verify with cache read/write token fields, never latency alone.

Deferred-tool mechanics

HTTP request: search tool + every full deferred definition
initial model context: search tool + non-deferred tools only
search result: tool_reference(name)
continued model context: old prefix + expanded selected schema

Preserve the provider's search blocks in history. Return a result only for the discovered client tool_use, never for the provider's server_tool_use.

Frequent traps

  • “The marked block alone is cached.” False: the whole preceding prefix is included.
  • “Stable content behind a changing marker will be discovered.” Only if a prior request actually wrote an entry within the 20-block lookback window.
  • “Automatic mode survives ordinary tool-array changes.” False: it automates endpoint placement, not prefix identity.
  • “All lazy loading must break the cache.” False: provider-native deferred tool search preserves the prefix by changing where schemas appear.
  • “A successful answer proves a cache hit.” Caching affects cost and latency, not logical correctness.
