capability matrix

Verified against provider documentation on 16 July 2026. Capabilities must be keyed by provider adapter and model, not assumed from a common AI SDK tool type.

Harness rule

resolve concrete provider + model → read capability profile → choose tool-loading strategy → build provider request → execute

Do not choose a provider-native tool representation after wrapping several providers behind an opaque retry/fallback model. A fallback can change not only model quality but the valid request protocol.
