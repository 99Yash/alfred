Which host needs which capability?

Capability fit follows the host’s real-world possessions and responsibilities.

CapabilityLocal IDE/desktopHosted Alfred
ToolsCoreCore
Remote OAuthWhen using remote serversCore for remote servers
Tool-list changesUsefulUseful
RootsCore for workspace-aware serversOmit: no user-local filesystem
ResourcesUseful for files/docsSelective for addressable remote data
Resource subscriptionsUseful for live filesSelective for upstream change signals
PromptsUseful as commands/templatesUsually redundant with skills/workflows
SamplingUseful when client controls model accessOff by default: cost and trust expansion
ElicitationNatural with interactive UIPossible through Alfred UI, selectively
TasksUseful for long server workSelective; does not replace durable runs
stdioNatural for local serversOnly server-side trusted processes
Streamable HTTPFor remote serversNatural default

Filter: do not ask “does MCP support it?” Ask “which Alfred workflow needs it, which participant offers it, and what new authority does enabling it grant?”
