Both state and transcript are inserted into agent_runs with status=pending. BullMQ receives only the run id.

Frame 2 — the leased chat step enriches state

Load A

Hydrate attachments

Storage references in the transcript become bounded image bytes for model input.

Provider-only. Raw bytes are not checkpointed back into the canonical transcript.

Load B

Resolve user environment

Load timezone and integration availability. Build the human-readable connected-services catalog.

Timezone and connectedSummary are copied into run state so later steps reuse the same snapshot.

Load C

Preload likely tools

The latest user message is ranked once. Selected exact tools are added to state.activeTools.

This happens before tool schemas and the system prompt are finalized. A changed active set changes the earliest provider prefix.

Load D

Load artifact context

Safe generated artifact metadata is prepared for the system prompt. The selected artifact body becomes a lower-trust assistant-role reference.

Metadata is stored in run state; the body reference is request-only.

Frame 3 — assemble the three provider sections

ToolsExact active schemas: kernel + deterministic preloads. Sorted alphabetically; execute removed.after last tool

Request only<runtime_context> and selected artifact body are inserted as an assistant message immediately before the newest user message.
Newest userThe actual current ask remains the final message.after last message

The step hashes the completed system text into state.systemPromptHash. A later model step in the same run must reproduce that hash unless an explicit lifecycle seam, such as an artifact mutation, clears it.

Frame 4 — context guard, then cache decoration

Before inference, Alfred estimates the whole request against the selected model’s context window. If needed, it compacts old history while retaining a separate storage-safe continuation transcript. Only after that does AlfredAgent attach Anthropic’s 1-hour cache markers.

Frame 5 — what survives the answer?

ItemAfter direct completion
Final assistant responseAppended to the run transcript and persisted as a chat_messages row.
Connected summary, timezone, active tools, system hashPersisted in final run state for audit/replay of this run.
Hydrated image bytesDiscarded; storage references remain canonical.
Runtime context and artifact body referenceDiscarded from the checkpoint; reconstructed when needed.
Provider cache entryProvider-owned, temporary, and only a performance optimization.

Cache nuance: the tool and system prefixes can be reused across runs when identical. The full message-prefix breakpoint is especially useful inside a contiguous tool loop because the runtime timestamp is deliberately held stable there. After a real park, Alfred clears that time anchor and rebuilds the request with fresh time.
