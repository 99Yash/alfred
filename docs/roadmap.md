# Alfred Roadmap

The app is still pre-release, but the build is live and moving in public. This
is the short path from working personal tool to a release-ready assistant.

Source: https://git.new/alfred

## 01 — Now: Private working build

Email triage, todo suggestions, durable runs, approvals, memory, and daily
briefing foundations are live enough to test against real data.

## 02 — Next: Reliability and release polish

Tighten the open-loop model, finish the briefing surface, harden action gates,
clean up OAuth reconnects, and make the inbox/todo flows feel boringly
dependable.

## 03 — Release: Public-facing rewrite

Rewrite the AI-drafted field notes by hand, publish the finished product story,
and ship Alfred with the code and roadmap in sync.
